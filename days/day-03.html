Section 03 — The Most Important Concept

The n8n Data Model — The JSON Items Array

Every single piece of data in n8n, at every stage, has this exact shape. Understanding this is non-negotiable. This is the foundational rule that separates those who understand n8n from those who merely use it.

The Anatomy of Every Item in n8n 
[
  {
    "json": {
      // Your actual data lives here
      "yourField": "yourValue",
      "anotherField": 42
    },
    "binary": {
      // Optional: files encoded as Base64
      "optionalFile": {
        "data": "base64...",
        "mimeType": "image/png"
      }
    },
    "pairedItem": {
      // Links this item back to its origin
      "item": 0,
      "input": 0
    }
  }
]

Why an Array and Not a Single Object?

Because n8n is built for bulk processing. If your trigger fetches 500 emails, it produces 500 items. Every subsequent node automatically loops through all 500 — you write your transformation logic once and it runs on every item. This is called implicit looping at the engine level, and it is the single biggest productivity feature in n8n vs. hand-coded scripts.

items[]

Every node receives an array. Even a single object is wrapped. This enables batch processing and mapping.

Array Wrapper

json

Your actual payload. All fields, nested objects, and arrays live inside this key. Accessed with {{ $json.fieldName }}.

Data Payload

pairedItem

Tracks lineage across nodes. Critical for error tracing and understanding which input produced which output.

Lineage Tracking

binary

Optional key for file data (images, PDFs, CSVs). Stored as base64 with metadata like filename and mimeType.

File Data
Section 04

The n8n Node Taxonomy

Every node in n8n falls into one of these categories. Knowing the taxonomy helps you choose the right tool for every stage of your pipeline.

Node Category Purpose Examples
Trigger Entry point — starts the workflow Manual, Schedule (Cron), Webhook, App Events
Action Performs work in an external system HTTP Request, Gmail, Postgres, Slack
Transform Reshapes or filters the items array Set, Code, Edit Fields, Filter, Limit
Logic/Flow Controls routing and branching IF, Switch, Merge, Loop Over Items, Wait
Sub-workflow Calls another workflow Execute Workflow, Sub-workflow Trigger
Utility Housekeeping operations Aggregate, Sort, Summarize, Remove Duplicates

The Execution Algorithm — Step by Step

When a workflow runs, the n8n engine executes this precise sequence. Understanding this algorithm is what separates engineers from button-clickers.

1
Trigger Event
Manual click, cron tick, incoming webhook, etc.
2
Create Execution
Record in DB with unique ID
3
Resolve Order
Respects connections and branches
4
Execute Nodes
Receive → Evaluate → Execute → Output
5
Mark Complete
SUCCESS or ERROR with log
The Execution Algorithm — Detailed 
// 1. TRIGGER EVENT fires (manual click, cron tick, incoming webhook, etc.)
// 2. ENGINE creates an "Execution" record in the database with a unique ID
// 3. ENGINE resolves the execution order (respects node connections and branches)

// 4. For each node in order:
for (node of workflow.nodes) {
  // a. Receive the items array from the previous node's output
  const inputItems = previousNode.output;

  // b. Evaluate all expressions ({{ $json.x }}, {{ $now }}, etc.) per item
  const evaluatedItems = inputItems.map(item => evaluateExpressions(item));

  // c. Execute the node's core logic (API call, transformation, conditional, etc.)
  const outputItems = node.execute(evaluatedItems);

  // d. Produce a new items array as output
  node.output = outputItems;

  // e. Store input + output in the execution log (for debugging)
  executionLog.store(node.id, inputItems, outputItems);
}

// 5. On error → Error Workflow triggers (if configured)
// 6. Execution record marked SUCCESS or ERROR with timestamp and duration
// 7. Execution log pruned per your retention settings

Critical Insight

Every arrow in n8n carries an array of JSON items — never a single value. This is the foundational rule. If you internalize nothing else from this course, internalize this.

Section 06

n8n Expressions — How to Access Data

n8n uses double curly braces {{ }} for dynamic values, evaluated at runtime inside a sandboxed JavaScript engine. Any standard JS string/array/object method is available.

Expression Reference 
{{ $json.fieldName }}              // Current item's data
{{ $json.user.email }}             // Nested field access
{{ $node["HTTP Request"].json.id }} // Data from a specific named node
{{ $now.toISO() }}                 // Current timestamp (ISO format)
{{ $itemIndex }}                   // Index of current item (0, 1, 2...)
{{ $runIndex }}                    // Which execution loop run we're on
{{ $json.title.toUpperCase() }}    // JavaScript methods work inline
Variable Type Description Example
$json Object The json payload of the current item {{ $json.id }}
$input Object All items received by the current node {{ $input.first().json.id }}
$items Function Get items from a specific previous node {{ $items("Webhook").length }}
$now Luxon DateTime Current timestamp {{ $now.toISO() }}
$today Luxon DateTime Start of today (00:00:00) {{ $today.toISO() }}
$workflow Object Workflow metadata (id, name, active) {{ $workflow.name }}
$execution Object Execution metadata (id, mode, startedAt) {{ $execution.id }}
$env Object Environment variables {{ $env.API_KEY }}
$node Object Access output from any previous node {{ $node["Set"].json.email }}

Fully managed SaaS. Zero infrastructure overhead. Starts at $20/month. Good for teams without DevOps.

Managed SaaS

Self-Hosted (Production)

Docker Compose with PostgreSQL + Redis. Reverse proxy (Nginx/Traefik). SSL. Backup strategy.

Enterprise

The Production Checklist

Before going live: Use PostgreSQL (not SQLite), enable Redis for queue mode, configure SMTP for notifications, set up volume backups, and use a reverse proxy with SSL.

What to Check

After executing, click on each node and inspect the Input and Output panels. Verify that the HTTP Request node received an empty array as input and returned an array with one item. Verify that the Set node received that array and returned a transformed array. This is how you diagnose what is happening inside a node.
