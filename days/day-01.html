Section 01

What n8n Is — And What It Is Not

n8n is a workflow automation platform built on Node.js. Unlike Zapier or Make, it gives you full control over the execution engine, data flow, and infrastructure.

Fair-Code Licensed

Source-available. You can self-host, modify, and audit every line of the execution engine.

Self-Hosted

DAG Execution

Workflows are Directed Acyclic Graphs. Nodes execute in topological order, not linearly.

Directed Graph

400+ Integrations

Built-in nodes for HTTP, databases, messaging, CRM, and AI — but you can also write custom nodes.

Extensible

Data Privacy First

Your data never leaves your infrastructure. No third-party processing unless you explicitly configure it.

Privacy

Section 03

Installation — Three Ways

Choose your deployment strategy based on your infrastructure, compliance requirements, and scale.

Docker (Recommended)

Single container with SQLite. Perfect for development, small teams, and rapid prototyping.

# Quick start
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  n8nio/n8n
Dev / Small Team

n8n Cloud

Fully managed SaaS. Zero infrastructure overhead. Starts at $20/month. Good for teams without DevOps.

Managed SaaS

Self-Hosted (Production)

Docker Compose with PostgreSQL + Redis. Reverse proxy (Nginx/Traefik). SSL. Backup strategy.

Enterprise

The Production Checklist

Before going live: Use PostgreSQL (not SQLite), enable Redis for queue mode, configure SMTP for notifications, set up volume backups, and use a reverse proxy with SSL.

Section 04

Your First Workflow — The Anatomy

Every workflow has exactly one trigger and one or more action nodes. Understanding the data structure that flows between them is critical.

Data Structure — What flows between nodes 
// n8n data is always an array of items
[
  {
    "json": {
      "id": 1,
      "name": "Ahmed",
      "email": "ahmed@example.com"
    },
    "pairedItem": { "item": 0 }
  }
]

Critical Rule #1

Every node receives an array and returns an array. Even if your data is a single object, n8n wraps it in items[]. This is why expressions like {{ $json.name }} work — n8n automatically maps over the array for you.
